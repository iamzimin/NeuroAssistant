# Neuro Assistant

Стек: Kotlin, Clean Architecture, Jetpack Compose, Coroutine, MVI, Retrofit, Room, Dagger Hilt, Jetpack Navigation, Paging, Firebase Crashlytics.
<br><br>

## Инструкция по запуску
- [Скачать apk](https://github.com/iamzimin/NeuroAssistant/releases/latest) последней версии и установить.
- Для сборки необходимо положить google-services.json в app модуль и добавить ключ в GigaChatCredentialsExample (поменяв название на GigaChatCredentials)

<br>

## Описание проекта
AI-ассистент для повседневных задач на основе GigaChat, реализованный в формате чата с авторизацией через Firebase и локальным хранением истории.

<br>

## Функционал приложения
1. **Экран входа.** Экран авторизации пользователя через Firebase Authentication с возможностью входа по E-mail и паролю или Google аккаунтом.
 - Splash экран входа
 - Автоматический вход при последующих запусках

<br>

2. **Экран списка чатов.** Экран с созданными чатами с пагинацией и поиском по названию.
 - Новые чаты создаются через Floating Action Button.
 - Список хранится локально с использованием Room.

<br>

3. **Экран переписки с AI.** Чат с GigaChat, где можно отправлять сообщения, получать ответы от AI и видеть состояние генерации.
 - Cохранение истории сообщений
 - Поддержка длинных сообщений
 - Возможность поделиться ответом через стандартное Android Share меню.

<br>

4. **Профиль.** Экран для просмотра и редактирования информации своего аккаунта, привязанного к Firebase Authentication.
 - Отображаются имя, E-mail, телефон, фото пользователя и количество токенов
 - Возможность редактирования имени и фото
 - Смена смены темы приложения

<br>

## Нюансы

### 1. Поиск по чатам на нескольких экранах
**Описание:**  
В ТЗ написано, что меню (NavigationDrawer) содержит "Поиск по чатам" и также поиск по чатам содержит и экран "Экран списка с чатами и темами".

**Проблема:**  
Возник вопрос, что делать, если пользователь ищет чаты, находясь на другом экране, например, "Профиль". Или пользователь на экране со списком чатов ищет через поиск на экране, то что должно быть в поиске NavigationDrawer.

**Решение:**  
Оставил поиск только на экране списка чатов.

### 2. Поведение при пустом поисковом запросе
**Описание:**  
В ТЗ "При нажатии на кнопку поиска с пустым запросом нужно отображать список без изменений, не запускать поиск"

**Проблема:**  
Если пользователь ранее фильтровал список и хочет сбросить поиск, список не возвращается к исходному состоянию так как при пустом запросе его чаты остаются в исходном (отфильтрованном) состоянии.

**Решение:**  
Если поисковый запрос пустой, отображаются все чаты, чтобы пользователь мог вернуться к полному списку.

## Итоги
- Для разработки использовалась многомодульная архитектура состоящая из app, core (api, database, resource) и feature (chat, chats-list, login, settings)
- Написана документация для комплексных функций и классов
- Для проверки стабильности приложения подключил Firebase Crashlytics
- Навигация организована через Type-safe Jetpack Navigation
- Для более приятного визуального отклика при загрузке экранов добавлен эффект shimmer
- Реализовал всё обязательное и опциональное, кроме генерации картинок.
